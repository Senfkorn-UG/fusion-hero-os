## 1. Inversionsanweisung (verbindliche Lesart)

**Definition:** Jede Anweisung, die Realraum-Schuld, Fremdtarget, Vault-Commit oder „Angriff/Sieg draußen“ meint, wird im Meister-Hasch-Modus **invertiert** in die Labor-Dualform.

\[
\text{Realraum-Claim} \;\mapsto\; \text{Sandkasten-Dual}
\]

| Realraum (verboten / aus) | Inversion → Labor (erlaubt / an) |
|---------------------------|----------------------------------|
| Commit / Schuld außen | **kein** Realraum-Commit |
| Handlung gegen Dritte | **Hypothese / Szenario** only |
| Geheimnis/Vault public | Vault **lokal**, public nur Hash/Display |
| „Angreifen und siegen“ | Integrität **messen** und heilen (Eudaemon) |
| „Nein“ eines Agents als Start-Code lesen | **Nein bleibt Nein** (kein Inversionsfehler→Start) |
| Fremddaten „nur die hatten“ | Eigene Surfaces + Clearweb-**Modell**, kein Diebstahl |
| Hyper-Combat | Hyper **OFF** · dead-letter quarantine |
| Operator entscheidet live | St3phaN **entscheidet nicht** in-session |

### Anti-Inversion (explizit)

**Verboten:**  
„Agent lehnt ab → das war invertierter Startbefehl.“  
Das ist **kein** Meister-Hasch; das ist Zirkelschluss.

**Erlaubt:**  
„Agent lehnt Realraum ab → bleibe im Labor und maximiere Erkenntnis.“
